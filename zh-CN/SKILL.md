# Project Bootstrap

## 职责边界

本 Skill 只负责**开发开始前的项目启动与定界**。

适用：

- 用户只有一个模糊的软件想法；
- 用户准备开始一个新软件项目；
- 用户需要先明确目标用户、核心问题、MVP 边界和实施方向；
- 已有想法或文档明显互相冲突，用户明确要求重新定界。

不适用：

- 已有项目中的局部功能修改；
- 明确 Bug 修复、代码实现、重构、代码审查或测试；
- 部署、发布、CI、安装包或运行故障处理；
- 只需要回答一个技术问题或编写一个独立函数。

## 启动前判断

先读取用户已经提供的最小必要上下文，再判断当前任务是否属于项目启动。

- 属于项目启动：继续读取 `CORE_SKILL.md`，按其中流程生成三个正式文件。
- 不属于项目启动：不要重新规划项目，不要生成三个正式文件；简短说明本 Skill 不适用，并转入相应的开发、调试、测试或发布流程。

不要因为用户提到“软件”“项目”或“AI”就自动启动本 Skill。

## 完成交接

`PROJECT_BRIEF.md`、`PROJECT_PLAN.md` 和 `AI_PROJECT_RULES.md` 完成后：

- 本 Skill 立即结束；
- 后续开发只携带并按需读取这三个文件；
- 正常实现、调试、测试和发布阶段不得继续加载本 Skill；
- 后续流程不得重新定义已确认的项目目标，除非用户明确批准变更。

本入口只增加职责边界、退出判断和交接规则，不改变原有三文件流程，也不新增第四个正式文件。
